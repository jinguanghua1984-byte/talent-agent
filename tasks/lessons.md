# Lessons

- 2026-05-24：脉脉 campaign 的 `account_day_page_guardrail` 是账号级执行护栏，不是业务需求硬上限；当用户明确换号并解除旧 500 上限时，必须同步更新 run-policy、后续 resume plan 和任务账本，不要继续用旧账号剩余额度截断 follow-up wave，但登录/验证码/403/429/432/非 JSON/模板漂移等平台安全停机规则仍然不变。
- 2026-05-24：实现或验收带有“自适应/动态”语义的脉脉 campaign 时，不能只验证评分函数、状态文件和报告产物；必须验证 live gate/orchestrator 是否实际消费 `next_page`/`adaptive-unit-state` 生成后续执行计划，并用 raw 页分布证明存在 page 3-N，否则只能称为 probe-only 后置评估。
- 2026-05-18：工作台里的默认说明文字不要复用日志/代码块样式，尤其不要用深色背景包裹普通提示；说明文本应使用浅色 hint 样式，日志列表、请求预览和提示条也要共享同一套浅色 surface token，避免同一界面出现两套日志背景。

- 2026-05-18：破坏性或全局数据操作（如清除全部数据）如果作用域是整个工作台，就应放在全局 topbar/action 区，而不是某个标签页面板内；图标风格也要跟主 UI token 一致，避免用红色/垃圾桶这类暗示删除文件的图标造成视觉和语义错位。

- 2026-05-18：扩展从 popup 迁到常驻工作台后，业务页内容脚本不要再挂悬浮窗或重复展示状态；页面里已有统计卡片时，标题区和 badge 不要再显示同一组请求/人选/详情数字。进度展示也要统一口径：自定义抓取应显示目标页数，不要启动时用 API 总页数、运行中再切到目标页数。

- 2026-05-18：Promise 链里前一个 `.then(existingContacts => ...)` 的参数不会自动延续到后一个 `.then()`；如果后续回调还要用这份数据，必须先写入外层作用域变量，或显式返回结构化对象再解构。

- 2026-05-18：Chrome Side Panel 打开不能靠默认参数或先写状态再异步 open；必须同步拿到 `windowId`/`tabId`，并优先在同一用户动作链里直接调用 `chrome.sidePanel.open(...)`，否则会回退成新标签页或直接失败。

- 2026-05-13：文档写到 CLI 参数时，必须同步检查 `argparse` 是否实际暴露该参数，并补 CLI 层测试；记录任务进度时只追加当前任务必要内容，不要把无关历史段落纳入本次 diff。

- 2026-05-13：当用户明确说“不需要某个 MCP / 插件 / 工具”时，不要继续沿修复该工具的方向推进；应立即切换为禁用或移除启动注册，并清理自己刚创建的临时补丁文件，验证启动配置不再加载该工具。

- 2026-05-12：当用户要求“验证实施计划中每个环节的技术可行性”时，不能把数据导入/本地评分链路已验证等同于浏览器端无人执行已验证；涉及平台搜索、CDP、扩展自动点击、导出下载、请求字段语义的环节必须逐项给出证据、未验证风险和 Plan B。

- 2026-05-13：脉脉企业端登录态很脆弱；即使标记为“只读健康检查”，也不能主动 `page.goto()`/刷新企业端人才银行 URL 来复核登录态。后续只允许用户手动打开目标页，我只读取已经稳定打开的现有页；如果没有现有人才银行页，停止并请用户打开，不自动导航。

- 2026-05-13：脉脉人才银行页不仅会因主动导航/刷新失效，也可能因扩展 automation 页切换、CDP 激活标签、自动化消息调用等看似“非导航”的操作触发平台安全机制并自动关闭页面。后续任何真实脉脉门禁复跑前，必须把“会改变活动窗口/标签/焦点或调用扩展后台的操作”视为有风险；没有用户明确授权，不再自动打开 automation 页、不切换 tab、不调用 `startDetailBatch`。

- 2026-05-13：分析脉脉自动化失败时，不要把 `Receiving end does not exist` 当成首要根因；它可能只是平台安全机制已触发、人才银行页已关闭/登出后的下游症状。根因分析要前移到“自动化路径中哪个行为先触发安全机制”，重点比较手动路径与自动化路径在窗口/焦点、sender、调用节奏、CDP 附着和扩展后台消息上的差异。

- 2026-05-13：当一个数量级门禁已经通过，例如 D3 的 30 人详情，不要在“下一步”默认再次生成 30 人任务包；应检查已验证规模和用户目标，直接放大到下一个有业务意义的集合，例如全部 A 档且尚无 `maimai_detail_capture` 的候选人。

- 2026-05-11：用户询问“方案文档是否设计完整”时，评审对象是文档的需求覆盖、架构闭环、风险与验收标准，不等同于检查代码是否已经实现；只有用户明确问“是否落地/是否实现/代码状态”时才切到实现核对。
- 2026-05-10：生成包含中文的报告时，不要把中文固定文案直接写进 PowerShell here-string 再交给 Python；该链路可能把非 ASCII 文案写成 `?`。应使用 UTF-8 文件模板、`apply_patch`，或在脚本中用 Unicode 转义构造文案，并用 `utf-8-sig` 写出面向 Windows 查看器的 Markdown。
- 2026-05-10：即使最终用 Python `utf-8-sig` 写文件，只要 Python 源码本身来自 PowerShell here-string 的中文字面量，仍可能提前乱码；面向用户的中文 Markdown 固定文案必须改用 `apply_patch` 模板或 Python Unicode 转义，并在写后读取首行校验。
- 2026-05-10：Chrome 扩展的 match pattern `*://maimai.cn/*` 不覆盖 `www.maimai.cn` 等子域。脉脉详情页捕获异常为 0 时，先检查 `host_permissions` 和 `content_scripts.matches` 是否包含 `*://*.maimai.cn/*`，再让用户重载扩展和刷新页面。
- 2026-05-10：Chrome 扩展重载后，旧页面中的 content script 会残留并抛 `Extension context invalidated`。修改扩展时要提示用户重载扩展后刷新业务页面；content script 中所有 `chrome.runtime.sendMessage` 也要包一层 try/catch 和 `chrome.runtime.id` 检查。
- 2026-05-10：`maimai-scraper` 的导出按钮如果优先走 IndexedDB 分页导出，会把被动拦截的详情请求漏掉；详情补全必须使用完整导出，同时包含 `contacts`、`details`、`requests`。清除数据也必须同步清 `PagerDB`，否则旧联系人列表会抢占导出结果。
- 2026-05-14：生成搜索 API 说明书时，不要只在口头摘要里挑少数筛选项；必须在机器 JSON 和 Markdown 中全量保留所有观测字段，同时单独分出关键字段、优先确认筛选字段、次级筛选字段、模板保留字段和其他字段。`search.query_relation` 语义已确认：`0=AND`、`1=OR`。
- 2026-05-14：用户说“另一个 Codex 账户继续”时，不要默认理解为换机器；先区分同机同工作区换账号、同机不同工作区、跨机器迁移。若是同机同工作区，优先依赖本地文件交接，不需要打包传输或迁移浏览器 profile。
- 2026-05-14：脉脉搜索字段 `major` 是专业名，不是职位名；`degrees` 学历编码为 `0=专科`、`1=本科`、`2=硕士`、`3=博士`、`100=自定义范围`，预设“本科及以上”仍需结合 `only_bachelor_degree` 区分“不限/只看统招本科”。
- 2026-05-14：脉脉搜索年龄字段已确认是范围筛选；参数统一为 `min_age/max_age`，不要实现或文档化 `age_min/age_max` 别名。
- 2026-05-14：当用户要求把寻访/采集目标按数量级放大时，不能只修改 KPI 数字；必须同步调整任务拆分粒度、wave/checkpoint、raw 原子落盘、resume 参数、import/apply ledger、停止规则和报告 funnel，否则长时任务中断后会丢数据或重复写入。
- 2026-05-14：AI Infra V2 寻访条件中，毕业院校是硬门槛：只看 985、211、QS Top500 或海外 Top500；专科和非重点院校不看。年龄默认只抓 `24-40`，其中 `24-35` 最佳、`35-40` 第二梯队，`40+` 淘汰；文档和实现计划都要同时反映搜索参数与评分分层。
- 2026-05-15：长时搜索任务如果因 429/403/验证码/登录/非 JSON/模板异常等条件中断，只记录中断点、中断原因、已完成页级 raw、continuation plan 和页面健康状态；不要基于不完整 wave 继续导入、apply、评分或生成评审草稿。只有该 wave 后续补齐并通过 clean dry-run 后，才继续 pipeline 后续步骤。
- 2026-05-15：脉脉 AI Infra V2 真实列表搜索当前观测到每日请求限额约 500 次；后续继续按 wave 粒度执行，不再为了降低验证码/限流而在 wave 内主动拆 50 页切片。已观察到验证弹出更像按接口执行总量判断，切片没有减少熔断次数；熔断后仍只记录断点、原因、已完成页级 raw 和 continuation plan，等待人工处理后继续。
- 2026-05-15：脉脉列表搜索的 API 级验证码可能表现为 HTTP 429；记录中断原因时不能只写限流，要检查 responseData/rawPreview/页面状态里是否有验证码或安全验证证据。用户人工通过验证码后，才允许从 continuation plan 继续。
- 2026-05-16：当用户把新能力明确归入现有 campaign 子任务时，不要新开分支或新建独立 campaign；继续在当前 campaign 分支开发，真实抓取结果只写 campaign DB，主库 `data/talent.db` 仍保持只读边界。
- 2026-05-18：任务目录下的 XML 草稿如果只是本地过程产物，应保留本地并通过 `.gitignore` 忽略，不要直接删除；提交前先确认它是源稿还是业务资产。
- 2026-05-19：设计脉脉无人值守 campaign 时，必须把浏览器 bootstrap 写成一等阶段：自动用 `data/session/maimai-cdp-profile` 和 `--remote-debugging-port=9888` 启动 CDP 浏览器、加载扩展、打开脉脉主页，然后由人工登录、进入人才银行页并手动执行一次搜索；执行阶段仍不得自动导航/刷新/点击业务页面。
- 2026-05-19：业务入口 skill 的问题应先从用户提示词或 JD 自动抽取，只问缺失信息；问题里的“冷启动”“关键词包”“停止阈值”等生词必须附带具体解释。已有实践默认值不要重复追问，例如每日搜索请求预算 500、搜索 wave 不超过 50 页、详情只抓 A/B 档且每包不超过 100 人、交付为本地 MD/CSV + 飞书文档/多维表格。
- 2026-05-19：无人值守长任务的一版通知若用户要求飞书 IM，就不能写成“后续接入”；spec 必须列出 bot/user 身份、scope、chat_id/user_id、测试消息、dry-run 和需要用户配合的配置步骤。
- 2026-05-19：脉脉无人值守 campaign 里“每日请求预算 500”必须明确写成“每日搜索请求预算 500”，只统计列表搜索请求，不包括详情请求；详情请求另按 pack 上限、health gate、请求间隔和平台安全停机规则控制。
- 2026-05-20：当用户明确要求“无人值守” campaign 时，搜索计划确认后不应反复让用户 check/apply，也不要再提示手动启动浏览器；run-policy 应覆盖 Campaign DB clean dry-run 自动 apply、列表后自动粗筛、详情后自动详评/精排和飞书交付包发布。只在登录/验证码/安全页/429/非 JSON/模板漂移等平台阻断、主库写入或策略异常时停下来。
- 2026-05-22：当通用 campaign 里出现 `AI Infra` 等首次样板任务痕迹时，不要只当作交付文案问题；优先沿脚本、模板、配置和报告生成链路追踪是否存在本应按 JD 动态生成却沿用样板数据的参数残留，并区分对搜索、评分、详情和交付各阶段的真实影响。
- 2026-05-23：JD talent delivery 不能把 campaign `strategy.json` 或历史 `*rank*.json` 当作必须前置；标准路径必须能从 JD 生成的 `scorecard.json` 和只读 `data/talent.db` 独立完成匹配、精排、质量门禁和飞书交付，campaign artifact 只能作为可选参考或排障输入。
- 2026-05-23：JD talent delivery 的飞书 IM 完成通知必须是 workflow 固定步骤，不是任务后人工补发；通知模板要落盘并包含任务执行结果、成果物清单、Wiki 目录链接和推荐报告摘要。外联 Sheet 不能再通过 `drive +import --type sheet` 导入 CSV，必须走 `sheets +create` + UTF-8 JSON `sheets +write`，把乱码预防放在发布机制里，而不是发布后靠校验修复。
- 2026-05-24：自适应扩页的回归测试不能把 page 3 预先写进 wave plan；必须让 `max_pages=probe_pages`，再验证 live gate 依据页质和 `unit_max_pages` 在执行中实际请求 page 3-N。
- 2026-05-25：长时无人值守任务里的“中断后关机”是一次性运行指令，不是永久策略；用户后续要求取消时，必须先查杀/确认无 `Stop-Computer`、`shutdown.exe` 或延迟关机进程，再继续恢复任务，并把 `shutdown_instruction_active=false` 写回 campaign 状态。
- 2026-05-25：用户要求同步 campaign DB 到主库时，必须优先按用户指定的 campaign 路径收窄范围；不能把 `data/campaigns/**/talent.db` 默认扩展为全部目录。若范围被用户更正，先停止残留全量任务，再只处理指定 DB。
