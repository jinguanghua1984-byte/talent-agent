# 脉脉人才抓取器

自动化在脉脉平台上搜索、筛选和抓取候选人信息的技能。

## 核心能力

1. **登录验证** - 等待用户完成手机验证码登录，自动检测登录状态
2. **条件搜索** - 支持 12 个搜索条件，动态映射到页面控件
3. **智能筛选** - 基于语义匹配的自定义筛选规则
4. **并发抓取** - 使用 subagent 并行获取详情页信息
5. **结果导出** - 生成带统计信息的 Excel 报告

## 工作流程

### Phase 1: 登录验证

```
目标：确保用户已登录脉脉，具备搜索权限

前置条件：agent-browser daemon 已安装
```

#### 1.1 检查 Daemon 状态

```powershell
.\adapters\claude-code\scripts\check-session.ps1
```

返回码说明：
- `0`：已登录，可直接使用
- `1`：daemon 未运行，需先运行 `.\adapters\claude-code\scripts\start-daemon.ps1`
- `2`：未登录，需在浏览器窗口中扫码

#### 1.2 首次使用流程

1. 运行启动脚本：
   ```powershell
   .\adapters\claude-code\scripts\start-daemon.ps1
   ```
2. daemon 会打开浏览器窗口
3. 访问 https://maimai.cn 并使用 APP 扫码登录
4. 登录成功后 session 自动加密保存
5. 后续使用无需重复登录

#### 1.3 Session 恢复

- daemon 重启后会自动恢复 session
- 如果 cookies 过期，需重新扫码登录
- 加密密钥存储在 `~/.agent-browser/maimai-key`

#### 1.4 登录状态验证

```powershell
# 检查登录状态
.\adapters\claude-code\scripts\check-session.ps1
```

**重要**：
- 不要尝试自动处理验证码，这是用户手动操作
- Windows 必须使用 headed 模式（`AGENT_BROWSER_HEADED=1`）

### Phase 2: 搜索条件收集与执行

```
目标：收集用户搜索条件并执行搜索

支持的搜索条件（12个）：
┌─────────────┬──────────────────────────────────────┐
│ 字段         │ 说明                                  │
├─────────────┼──────────────────────────────────────┤
│ 城市地区     │ 目标工作城市，支持多选                 │
│ 学历要求     │ 最低学历要求                          │
│ 工作年限     │ 工作经验范围                          │
│ 就职公司     │ 当前或曾就职公司                       │
│ 职位名称     │ 目标职位关键词                        │
│ 行业方向     │ 目标行业领域                          │
│ 毕业学校     │ 院校名称关键词                        │
│ 专业         │ 专业名称关键词                        │
│ 性别         │ 男/女/不限                           │
│ 年龄         │ 年龄范围                              │
│ 期望月薪     │ 薪资范围                              │
│ 家乡         │ 籍贯筛选                              │
└─────────────┴──────────────────────────────────────┘

执行步骤：
1. 向用户展示搜索条件列表
2. 逐个询问用户需要填写的条件（跳过不需要的）
3. 将用户输入映射到页面控件
4. 执行搜索操作
5. 确认搜索结果页面加载成功
```

**控件映射参考**：参见 `references/maimai-fields.md`

### Phase 3: 筛选策略收集

```
目标：收集用户的筛选规则，用于过滤候选人

执行步骤：
1. 告知用户：现在设置筛选策略，不符合条件的候选人将被跳过
2. 交互式收集筛选规则：
   - 提示用户输入规则（自然语言描述）
   - 用户输入 'done' 结束
3. 解析并存储筛选规则

示例筛选规则：
- "近三年跳槽不超过2次"
- "学历本科及以上"
- "期望薪资在 25K-40K 之间"
- "有互联网大厂经历优先"
- "工作地点必须在上海或杭州"
```

**筛选规则处理（混合模式）**：

筛选规则不是页面控件，而是对抓取结果的后处理。采用混合匹配策略：

1. **结构化规则优先解析**：
   - 跳槽频率：解析为 `{type: 'job_hops', years: 3, max_count: 2}`
   - 学历要求：解析为 `{type: 'education', min_level: '本科'}`
   - 薪资范围：解析为 `{type: 'salary', min: 25000, max: 40000}`
   - 年龄范围：解析为 `{type: 'age', min: 25, max: 35}`

2. **复杂规则使用 LLM 语义判断**：
   - "有互联网大厂经历" → 检查工作履历中的公司知名度
   - "技术栈匹配" → 检查职业标签与职位要求的相关性
   - "稳定性好" → 综合评估工作履历时长

3. **匹配执行流程**：
   ```
   FOR 每个候选人:
     1. 尝试将规则解析为结构化条件
     2. 能结构化的 → 直接对候选人数据匹配
     3. 无法结构化的 → 调用 LLM 语义判断
     4. 记录匹配结果和淘汰原因
   END FOR
   ```

### Phase 4: 候选人抓取

```
目标：循环抓取所有候选人信息，直到最后一页

数据字段分布：
┌─────────────────┬──────────┬──────────┐
│ 字段             │ 列表页   │ 详情页   │
├─────────────────┼──────────┼──────────┤
│ 姓名             │ ✓        │ ✓        │
│ 活跃度           │ ✓        │ ✓        │
│ 年龄             │ ✓        │ ✓        │
│ 工作年限         │ ✓        │ ✓        │
│ 学历             │ ✓        │ ✓        │
│ 工作地点         │ ✓        │ ✓        │
│ 期望工作地点     │ ✓        │ ✓        │
│ 期望薪资         │ ✓        │ ✓        │
│ 期望职位         │ ✓        │ ✓        │
│ 职业标签         │ 部分      │ ✓ 完整   │
│ 工作履历         │ 部分      │ ✓ 完整   │
│ 教育履历         │ 部分      │ ✓ 完整   │
└─────────────────┴──────────┴──────────┘

抓取流程：
┌─────────────────────────────────────────────────────┐
│ FOR 每一页列表:                                      │
│   1. 解析列表页候选人卡片                             │
│   2. 提取列表页可见字段                               │
│   3. FOR 每个候选人 (并发 2-3 个):                    │
│      a. 打开详情页                                   │
│      b. 提取完整信息（职业标签、工作履历、教育履历）   │
│      c. 关闭详情页                                   │
│   4. 应用筛选策略，过滤不满足条件的候选人             │
│   5. 记录淘汰原因                                    │
│   6. 保存通过的候选人                                │
│   7. 检查是否有下一页                                │
│   8. 如有下一页：翻页并继续                          │
│ END FOR                                             │
└─────────────────────────────────────────────────────┘
```

**并发策略**：
- 使用 Agent 工具创建 subagent
- 每个 subagent 负责获取一个候选人的详情
- 同时运行 2-3 个 subagent
- 主线程负责协调和汇总

**反爬策略**：参见 `references/anti-scraping.md`

### Phase 5: 结果输出

```
目标：生成抓取摘要和 Excel 报告

输出内容：
1. 控制台摘要：
   - 总抓取数量
   - 通过筛选数量
   - 淘汰数量及原因分布
   - 抓取耗时

2. Excel 文件：
   文件名：candidates_脉脉_{时间戳}.xlsx

   Sheet 1 - 统计概览：
   ┌─────────────────┬────────┐
   │ 指标             │ 数值   │
   ├─────────────────┼────────┤
   │ 总抓取数量       │ XX     │
   │ 通过筛选数量     │ XX     │
   │ 通过率          │ XX%    │
   │ 抓取时间        │ 时间戳 │
   │ 搜索条件        │ ...    │
   │ 筛选规则        │ ...    │
   ├─────────────────┼────────┤
   │ 淘汰原因分布     │        │
   │ - 跳槽频繁       │ XX     │
   │ - 学历不符       │ XX     │
   │ - 薪资不符       │ XX     │
   │ - ...           │        │
   └─────────────────┴────────┘

   Sheet 2 - 候选人详情：
   每行一个候选人，包含所有字段
   工作/教育履历使用 JSON 字符串或合并单元格
```

## 工具依赖

- **Agent**: 创建 subagent 并行获取详情（必需）
- **Write/Bash**: 生成 Excel 文件（必需）

## 使用示例

```
用户：帮我从脉脉抓取 Java 开发候选人

Skill 响应：
[Phase 1] 正在打开脉脉页面...
检测到未登录状态，请完成手机验证码登录。
登录完成后请告诉我。

用户：登录好了

Skill 响应：
[Phase 2] 请输入搜索条件（不需要的条件直接回车跳过）：
- 职位名称：Java开发
- 城市地区：上海,杭州
- 工作年限：3-5年
- 学历要求：本科
- 期望月薪：25K-40K
- 其他条件...（跳过）

[Phase 3] 请输入筛选规则（输入 'done' 结束）：
规则1：近三年跳槽不超过2次
规则2：有大厂经历优先
done

[Phase 4] 开始抓取候选人信息...
第1页：找到 20 位候选人
  并发获取详情... 完成
  应用筛选：通过 8 人，淘汰 12 人
第2页：找到 20 位候选人
  ...

[Phase 5] 抓取完成！
总计：100 位候选人
通过：35 位
淘汰：65 位
  - 跳槽频繁：28 人
  - 无大厂经历：22 人
  - 其他原因：15 人

Excel 已保存至：candidates_脉脉_20260316_143022.xlsx
```

## 注意事项

1. **登录安全**：不要存储或传输用户的登录凭据
2. **合规使用**：仅用于合法的招聘目的，遵守脉脉用户协议
3. **频率控制**：严格遵守反爬策略，避免账号被封禁
4. **数据隐私**：抓取的候选人信息应妥善保管，不对外泄露
