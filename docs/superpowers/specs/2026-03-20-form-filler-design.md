# 脉脉搜索表单自动填充模块设计

> **状态**: 待审阅
> **日期**: 2026-03-20
> **关联**: maimai-scraper skill 重构

## 背景

### 问题

当前 maimai-scraper skill 的工作流只支持单组搜索条件，无法满足批量搜索场景。

### 目标

1. 支持从 Excel 读取多组搜索条件
2. 自动填充脉脉搜索页面的复杂表单控件
3. 支持每组条件配对应的筛选规则
4. 多组条件的抓取结果合并去重

---

## 整体架构

### 模块化重构

将 maimai-scraper 从单文件 skill 重构为模块化架构：

```
adapters/claude-code/skills/maimai-scraper/
├── SKILL.md                        # 主入口：编排工作流
├── modules/                        # 独立模块
│   ├── login/                      # M1: 登录验证
│   │   └── index.ts
│   │
│   ├── form-filler/                # M2: 表单填充（本次设计核心）
│   │   ├── index.ts                # 主入口
│   │   ├── reader.ts               # Excel 读取
│   │   ├── mapper.ts               # 字段→控件映射
│   │   ├── types.ts                # 类型定义
│   │   └── fillers/                # 各类控件填充器
│   │       ├── base.ts             # 基类
│   │       ├── text.ts             # 文本输入
│   │       ├── select.ts           # 下拉单选
│   │       ├── cascade.ts          # 级联选择
│   │       ├── tags.ts             # 标签输入
│   │       └── range.ts            # 范围滑块
│   │
│   ├── filter-setup/               # M3: 筛选设置
│   │   └── index.ts
│   │
│   ├── scraper/                    # M4: 候选人抓取
│   │   └── index.ts
│   │
│   └── exporter/                   # M5: 结果导出
│       └── index.ts
│
├── scripts/                        # 调试脚本
│   └── debug-form-filler.ts
│
└── references/
    ├── maimai-fields.md            # 页面字段映射
    └── form-controls-map.md        # 控件选择器配置
```

## 核心模块：form-filler

### 功能

读取 Excel 中的搜索条件，自动填写到脉脉搜索页面的表单。

### 支持的控件类型

| 控件类型 | 填充器 | 示例字段 |
|----------|--------|----------|
| 文本输入 | text | 关键词、职位名称、毕业学校、专业 |
| 下拉单选 | select | 学历要求、性别 |
| 级联选择 | cascade | 城市地区、行业方向 |
| 标签输入 | tags | 就职公司 |
| 范围滑块 | range | 期待月薪、年龄 |

### Excel 格式

**列名严格与脉脉页面搜索项名称一致**：

| 列名 | 类型 | 示例值 | 说明 |
|------|------|--------|------|
| 关键词 | 搜索条件 | Java,后端,架构 | 支持多关键词，逗号分隔 |
| 关键词模式 | 搜索条件 | AND / OR | AND=满足全部，OR=满足任一 |
| 城市地区 | 搜索条件 | 上海,杭州 | 多城市用逗号分隔 |
| 学历要求 | 搜索条件 | 本科 | |
| 工作年限 | 搜索条件 | 3-5年 | |
| 就职公司 | 搜索条件 | 阿里,腾讯 | 多公司用逗号分隔 |
| 职位名称 | 搜索条件 | Java开发 | |
| 行业方向 | 搜索条件 | 互联网 | |
| 毕业学校 | 搜索条件 | 985 | |
| 专业 | 搜索条件 | 计算机 | |
| 性别 | 搜索条件 | 男 | |
| 年龄 | 搜索条件 | 25-35 | 范围格式 |
| 期待月薪 | 搜索条件 | 25K-40K | 范围格式 |
| 家乡 | 搜索条件 | 浙江 | |
| 筛选规则 | 筛选条件 | 近三年跳槽不超过2次;有Spring经验 | 多规则用分号分隔 |

**规则**：
- 列名必须与上表完全一致（与页面搜索项名称保持一致）
- 第一行为表头
- 空单元格表示不填写该条件
- **关键词**：支持输入多个关键词，通过"关键词模式"列控制逻辑
- 多值字段用逗号分隔
- 筛选规则用分号分隔多条

**关键词模式说明**：

| 模式 | 含义 | 示例 |
|------|------|------|
| AND | 候选人必须同时包含所有关键词 | "Java,后端,架构" → 必须三者都有 |
| OR | 候选人包含任一关键词即可 | "Java,后端,架构" → 有其一即可 |
| 空 | 默认使用 AND 模式 | |

### 调试命令

| 命令 | 作用 |
|------|------|
| `--dry-run` | 只解析 Excel，不操作页面 |
| `--row 3` | 只调试第 3 行 |
| `--only 关键词,城市地区,学历要求` | 只调试指定字段 |
| `--step-by-step` | 每步暂停确认 |
| `--slow` | 放慢操作速度 |

### 输出报告

```
=== 表单填充报告 ===
Excel 文件: search-conditions.xlsx
总行数: 5
处理行数: 5

--- 第 1 行 ---
✓ 关键词: Java,后端 (模式: AND)
✓ 城市地区: 上海,杭州
✓ 学历要求: 本科
✗ 就职公司: 元素未找到

统计: 成功 4/5 行
```

## 执行模式

| 模式 | 命令 | 说明 |
|------|------|------|
| 完整模式 | `/maimai-scraper --file data.xlsx` | 登录 → 循环抓取 → 导出 |
| 仅填充 | `--fill-only` | 只填表单，不抓取 |
| 仅抓取 | `--scrape-only` | 手动填好后执行抓取 |
| 调试模式 | `--debug` | 单步执行 |

## 开发计划

| 阶段 | 内容 | 优先级 |
|------|------|--------|
| 1 | 实现 form-filler 模块（含 5 种填充器） | ★ 最高 |
| 2 | 实现 Excel 读取 + 字段映射 | 高 |
| 3 | 改造主 skill 为循环编排 | 高 |
| 4 | 实现结果合并去重逻辑 | 中 |
| 5 | 完善调试脚本和日志 | 中 |

---

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 页面 DOM 结构变化 | 控件选择器配置化，易于更新 |
| 反爬检测 | 操作间隔随机化，支持慢速模式 |
| Excel 格式不规范 | 健壮的解析 + 详细的错误报告 |
| 多条件组合查询无结果 | 记录无结果的条目，支持重试 |
