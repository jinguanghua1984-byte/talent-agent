# JD 推荐反馈填写指南

> 给业务、猎头和招聘同学使用：在 outreach 表里用一句话写清候选人反馈，让下一轮推荐更准。

---

## 这份反馈表是做什么的

系统会先根据 JD 推荐候选人，并生成一张 outreach 表。你看完候选人后，只需要填写最后的 `feedback_note` 一列。

这些反馈不是为了考核个人判断，而是为了回答几个问题：

- 哪些候选人真的值得推进？
- 哪些候选人看起来分高，但实际岗位不匹配？
- 是岗位画像理解偏了，还是评分和匹配需要调整？
- 下一轮推荐应该放宽、收紧，还是换方向？

填得越具体，下一轮推荐就越容易改准。

---

## 只需要填写一个字段

业务侧只需要填写 `feedback_note`。旧版的 8 个反馈列已经废弃，不再要求业务填写：`feedback_label`、`feedback_stage`、`reason_codes`、`hunter_note`、`contacted`、`submitted_to_client`、`interviewed`、`offer`。

系统后续会从 `feedback_note` 中解析内部标签和原因码。你不需要学习原因码，也不需要判断问题属于哪个技术阶段。

---

## 怎么写 feedback_note

每一行代表一个候选人。请在 `feedback_note` 里写一句自然语言，尽量同时说明：

- 总体判断：认可、待定或不认可。
- 判断理由：为什么合适或不合适。
- 下一步：要优先联系、再确认，还是不建议推进。

好的反馈示例：

```text
方向匹配，有大模型平台经验，建议优先联系。
```

不够好的反馈示例：

```text
还行
```

如果你不确定，也可以直接写“不确定”的原因，例如“看不出是否负责核心模块，需要再确认”。

---

## 示例

| 场景 | feedback_note |
| --- | --- |
| 认可 | 方向匹配，有大模型平台经验，建议优先联系。 |
| 待定 | 公司和标题相关，但看不出是否真正负责核心模块，需要再确认。 |
| 不认可 | 关键词命中了，但实际做销售支持，不是我们要找的大模型平台产品负责人。 |

---

## 常见写法

| 情况 | 可以这样写 |
| --- | --- |
| 很匹配 | 候选人做过大模型平台产品，近期经历和 JD 匹配，建议优先联系。 |
| 需要确认 | 标题相关，但简历没有写清是否负责核心模块，需要电话确认。 |
| 方向不对 | 做过 AI 项目，但主要负责销售支持，不符合平台产品负责人要求。 |
| 资历不符 | 背景相关，但目前更像执行层产品经理，和负责人岗位有差距。 |
| 信息过期 | 候选人方向匹配，但近期状态不清楚，需要先确认是否看机会。 |

## 常见问题

### 必须每个候选人都填吗？

不必须。优先填写你看过、判断明确、准备推进或明确不推进的人。

### 需要写固定格式吗？

不需要。自然语言即可。建议包含“是否认可 + 为什么 + 下一步”。

### 还需要填旧版 8 个反馈列吗？

不需要。旧版八列说明已经废弃。当前业务表只保留 `feedback_note`，系统会在后台解析成内部结构化字段。

### 反馈会直接发给候选人或客户吗？

不会。`feedback_note` 用于内部复盘和下一轮推荐校准。写的时候仍建议保持专业，不写无关个人评价。

### 后续系统会自动改人才库或评分卡吗？

不会。反馈只生成校准建议，不直接写入人才库，也不会自动修改评分卡或自动发布到知识库。
