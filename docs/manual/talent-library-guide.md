## 什么是人才库

人才库是你的私人候选人档案柜。你可以把从各个渠道收集到的候选人信息统一存放在这里，随时查找、筛选、打分，不用再翻 Excel、翻聊天记录。

**你能做什么：**

- 把脉脉、Boss 直聘等平台的搜索结果一键存入人才库
- 按公司、职位、城市、学历、工作年限等条件快速查找
- 粘贴一份职位描述（JD），让系统帮你找出最匹配的候选人
- 给候选人打综合分或匹配某个职位的分数
- 从脉脉抓取候选人的完整履历、教育、项目经历
- 同步你和候选人的微信聊天记录，方便回顾沟通历史
- 更新或删除候选人信息

**操作方式：** 直接用日常中文跟系统说话就行。比如："帮我找上海 8 年以上经验的 Java 候选人"。

## 把候选人录入人才库

### 什么时候用

你在脉脉、Boss 直聘等平台上搜索到了一批候选人，想把它们存到人才库里统一管理。或者你有一份候选人名单（Excel 导出的、别人发给你的），想批量导入。

### 怎么操作

直接告诉系统你要导入候选人，把候选人信息发给它。可以是一条一条发，也可以把整个搜索结果文件发过来。

**示例对话：**

> **你：** 把这批脉脉搜索结果导入人才库

> **你：** 导入这个文件里的候选人：data/output/maimai-search-result.json

系统会先给你看一个**预览报告**，告诉你：

| 结果 | 说明 |
|------|------|
| 新建 | 人才库里没有的，会新建档案 |
| 合并 | 人才库里已经有同一个人，会自动合并新信息 |
| 待确认 | 系统不确定是不是同一个人，需要你来判断 |
| 失败 | 数据不完整，无法导入的记录 |

**你需要做的：** 看一眼预览报告，确认没问题后说"确认导入"。

如果有"待确认"的候选人，系统会把它们列出来，让你判断是不是同一个人。比如：

> 这两位候选人可能是同一个人，需要你确认：
> - 张伟，阿里巴巴高级工程师，北京
> - 张伟，蚂蚁集团技术专家，杭州
> 是同一个人吗？

### 导入完成后

系统会生成一份导入报告，记录这次导入了多少人、合并了多少人、失败了多少，方便你以后回查。

## 查找候选人

### 什么时候用

你需要从人才库里找人的时候。比如客户说"帮我找几个有阿里巴巴背景的 Java 候选人"，或者你自己想看看库里有哪些上海的研发总监。

### 怎么操作

直接用中文描述你想找什么样的人。

**按条件查找的例子：**

> **你：** 找公司是字节跳动的候选人

> **你：** 查找上海的候选人，工作年限 8 年以上

> **你：** 找学历硕士以上、职位包含"总监"的候选人

> **你：** 找有推荐人信息的候选人

> **你：** 查看综合分 80 以上的候选人

**按关键词查找的例子：**

> **你：** 搜一下履历里提到"分布式系统"的候选人

> **你：** 全文搜索"供应链管理"相关的人

### 查看结果

查找结果会以表格形式展示：

| 字段 | 含义 |
|------|------|
| ID | 候选人在人才库中的编号 |
| 姓名 | 候选人姓名 |
| 公司 | 当前所在公司 |
| 职位 | 当前职位 |
| 城市 | 所在城市 |
| 年限 | 工作年限 |
| 学历 | 最高学历 |
| 数据级别 | 信息完整程度（basic=基础信息，detailed=详细信息） |
| 综合分 | 系统给的综合评分 |
| 来源 | 信息来源平台 |

### 翻页和导出

默认每次显示 20 人。你可以：

- 说"下一页"继续查看
- 说"导出结果"，系统会生成一份完整报告文件

## 用职位需求匹配候选人

### 什么时候用

客户给了你一个职位（JD），你需要从人才库里找出最匹配的候选人。不同于简单的查找，匹配会根据职位的具体要求（技能、经验、行业背景等）逐一比对每个候选人，给出匹配度和推荐理由。

### 怎么操作

**方式一：直接粘贴职位描述**

> **你：** 帮我用这个 JD 匹配人才库的候选人：
> 职位：高级 Java 开发工程师
> 公司：某互联网大厂
> 要求：5 年以上 Java 经验，有分布式系统开发经验，熟悉微服务架构……

**方式二：指定已有的 JD 文件**

> **你：** 用 data/jd/backend-lead.md 这个职位匹配候选人

### 匹配结果

系统会告诉你：

- **Top N 推荐候选人**：最匹配的前几名候选人
- **匹配理由**：每个候选人为什么匹配（比如"8 年 Java 经验，在美团做过微服务改造"）
- **关键差距**：候选人不完全满足的地方（比如"学历要求硕士，候选人为本科"）
- **风险点**：需要注意的问题（比如"候选人最近刚跳槽"）
- **置信度**：系统对匹配结果的把握程度

匹配完成后，系统会给每个候选人打一个"JD 匹配分"，方便你横向比较。

### 导出推荐结果

> **你：** 导出匹配结果

系统会生成报告文件。如果后续想补全推荐候选人的详细信息，可以直接说：

> **你：** 把匹配结果前 10 名的脉脉详情抓取一下

## 给候选人打分

### 什么时候用

你想对候选人的整体质量做一个评估，或者想知道某个候选人对特定职位的匹配程度。

### 两种评分

**综合评分** —— 评估候选人本身的素质

综合评分会从以下维度来考量：
- 信息完整度（简历越完整，越容易评估）
- 职业轨迹（晋升路径是否清晰合理）
- 公司背景（是否在大厂、知名企业工作过）
- 职级（当前职级和行业对标）
- 技能稀缺性（掌握的技能是否稀缺）
- 求职状态（是否在积极看机会）

> **你：** 给张伟打综合分

> **你：** 给所有综合分低于 60 的候选人重新评分

**JD 匹配评分** —— 评估候选人对某个职位的适合程度

> **你：** 帮候选人 ID 440 计算跟"高级 Java 开发"这个职位的匹配分

### 评分说明

- 系统打分会告诉你为什么给这个分数，不是凭空给的
- 如果候选人已有旧分数，系统会先展示旧分数和新分数的对比，等你确认后才更新
- 评分结果会永久记录，你可以随时查看历史评分

## 补全候选人详细信息

### 什么时候用

人才库里的候选人可能只有基本信息（姓名、公司、职位）。你想要更完整的信息——完整工作经历、教育背景、项目经验等。

### 怎么操作

告诉系统你想补全谁的信息。

> **你：** 抓取张伟的脉脉详情

> **你：** 补全候选人 ID 440 的详细信息

系统会去脉脉平台抓取这个候选人的完整履历信息，包括：

- 完整工作经历（每段经历的公司、职位、时间、工作内容）
- 教育背景（学校、专业、学历、时间）
- 项目经历（项目名称、角色、描述）
- 联系方式线索

### 批量补全

如果你有一批候选人需要补全信息，比如匹配出来的 Top 10 推荐人选：

> **你：** 把匹配结果前 10 名的脉脉详情批量抓取

系统会生成一个抓取清单，你需要配合 Chrome 浏览器里的脉脉工具来完成批量抓取。具体步骤：

1. 系统生成抓取目标清单文件
2. 你在 Chrome 浏览器中打开脉脉
3. 在脉脉工具中导入清单
4. 等待批量抓取完成
5. 导出抓取结果
6. 系统先展示预览报告（新旧信息对比），你确认后写入人才库

### 小批量手动补全

如果只需要补全几个人，可以用更简单的方式：

1. 在 Chrome 浏览器中打开脉脉候选人列表页
2. 按系统给你的清单逐个点开候选人详情
3. 脉脉工具会自动记录详情数据
4. 完成后导出数据，系统帮你写入人才库

## 更新候选人信息

### 什么时候用

你发现了候选人信息的变化或错误，需要更新。比如候选人换了新公司、拿到了新的联系方式、或者之前录入的信息有误。

### 怎么操作

直接告诉系统要更新什么。

**更新联系方式：**

> **你：** 更新张伟的手机号为 13800138000

> **你：** 给候选人 ID 440 添加邮箱 zhangwei@example.com

系统会先展示旧值和新值：

> 联系方式更新确认：
> - 手机号：原值"无" → 新值"13800138000"
> 确认更新吗？

**更新其他信息：**

> **你：** 张伟跳槽去了字节跳动，更新他的公司信息

> **你：** 候选人李明的学历应该是博士，目前写的是硕士，修正一下

### 批量更新

批量更新会先展示预览报告，确认后才执行。

## 同步微信聊天记录

### 什么时候用

你和候选人已经在微信上沟通过了，想把聊天记录保存到人才库里，方便以后回顾沟通历史、了解候选人意向变化。

### 怎么操作

告诉系统你要同步谁的聊天记录，以及时间范围。

> **你：** 同步候选人张伟的微信聊天记录，联系人是"张伟-阿里"，时间从 2026-04-01 到 2026-04-30

你需要提供：
- **哪位候选人**（姓名或 ID）
- **微信联系人是叫什么**（微信里显示的名字，或者群名）
- **时间范围**（起始时间和结束时间）

### 同步结果

系统会把聊天记录保存成文档，并在人才库里建一条索引记录：

- 候选人是谁
- 微信联系人叫什么
- 聊天时间范围
- 一共多少条消息
- 文档保存在哪里

默认不会展示聊天全文（保护隐私）。如果你需要查看：

> **你：** 展示张伟的微信聊天记录

系统会提醒你内容可能包含敏感信息，确认后再展示。

## 删除候选人

### 什么时候用

候选人信息录入错误、重复录入了同一个人、或者你不再需要保留某人的信息。

### 怎么操作

> **你：** 删除候选人张伟

> **你：** 删除候选人 ID 440

### 删除前确认

删除是不可逆的操作，系统会非常谨慎：

1. 先展示候选人的基本信息，确认你找对了人
2. 展示关联的所有数据（详细信息、评分记录、聊天记录等）
3. 要求你输入确认语句：`确认删除候选人 <编号>`

> 删除确认：
> - 候选人：张伟（ID: 440）
> - 关联数据：详细信息 1 条、评分记录 3 条、微信聊天 2 条
>
> 请输入"确认删除候选人 440"以完成删除。

只有你手动输入了确认语句，删除才会真正执行。

## 常见问题

### 问：我说的话系统没理解怎么办？

换个说法就行。比如"找上海做 Java 的人"和"查找工作城市为上海、职位包含 Java 的候选人"效果是一样的。系统偏好更具体的描述，越具体结果越精准。

### 问：导入的时候有些候选人显示"待确认"，是什么意思？

系统检测到人才库里可能已经有一个相似的人，但不确定是不是同一个人（比如同名但不同公司）。需要你来判断是不是同一个人，再决定是合并还是分开保存。

### 问：综合分是怎么算出来的？

综合分从多个维度评估：信息完整度、职业轨迹、公司背景、职级、技能稀缺性、求职状态。每个维度都有权重，系统会给出评分理由，你可以在评分报告中查看。

### 问：删除操作能撤回吗？

不能。删除是永久性的，候选人及所有关联数据（详细信息、评分记录、聊天记录索引）都会被清除。所以系统会在删除前让你确认两次。

### 问：候选人信息更新错了怎么办？

你可以再次使用更新功能修正信息。联系方式等字段的每次更新都会展示旧值和新值，方便你核对。

### 问：我只想看看某位候选人的完整信息，怎么做？

> **你：** 展示候选人张伟的详细信息

> **你：** 查看 ID 440 的完整档案

系统会展示这个候选人的所有已录入信息。
