## 为什么要同步聊天记录

做猎头经常需要和候选人在微信上沟通。时间一长，之前聊了什么就容易忘：

- 候选人上次说期望薪资是多少来着？
- 他对那个岗位有没有兴趣？
- 他什么时候方便面试？
- 他之前提到过有推荐人吗？

把聊天记录同步保存下来，好处是：

- **随时翻阅** — 不用再翻微信聊天记录往上滑半天
- **按候选人归档** — 每个人的聊天记录都和他在人才库里的档案关联在一起
- **方便交接** — 如果需要同事跟进，对方可以快速了解之前的沟通内容
- **记录关键信息** — 聊天中提到的联系方式、意向变化都有据可查

## 使用前需要准备什么

同步聊天记录需要三个条件：

| 条件 | 说明 |
|------|------|
| 候选人已在人才库 | 你要同步的那个人必须已经录入人才库。如果还没有，先用「导入候选人」功能录入 |
| 知道微信里的名字 | 你在微信里看到的那个人（或群）叫什么名字，需要告诉系统 |
| 确定时间范围 | 你想同步哪一段时间内的聊天，比如"最近一个月"或"4月1号到4月30号" |

## 同步聊天记录

### 基本操作

直接用日常中文告诉系统你要同步谁的聊天、时间范围是什么。

**你需要告诉系统三件事：**

1. **候选人是谁** — 说姓名、说编号都行
2. **微信里叫什么** — 你在微信通讯录里看到的名字，或者群聊的名字
3. **从什么时候到什么时候** — 起始和结束的时间

**示例：**

> **你：** 同步候选人张伟的微信聊天记录，联系人是"张伟-阿里"，时间从 2026-04-01 到 2026-04-30

> **你：** 把我和候选人 ID 440 的微信聊天同步一下，微信名叫"张三"，最近两周的

> **你：** 同步候选人李明在"字节架构师交流群"的群聊，时间是 2026 年 3 月整月

### 系统会做什么

收到你的指令后，系统会：

1. **确认候选人** — 找到你在人才库里说的那个人，展示给你看是不是找对了
2. **展示同步计划** — 告诉你即将同步的范围：

   ```
   即将同步：
   - 候选人：张伟（ID: 440）
   - 微信联系人：张伟-阿里
   - 时间范围：2026-04-01 至 2026-04-30
   - 输出目录：data/wechat-timelines/
   ```

3. **执行同步** — 从微信导出聊天记录，保存为文档
4. **写入索引** — 在人才库里记录一条索引，方便以后查找

### 同步完成后

系统会给你一份同步摘要：

| 信息 | 说明 |
|------|------|
| 候选人 | 这份聊天记录属于谁 |
| 聊天名 | 微信联系人或群的名字 |
| 时间范围 | 聊天的起止时间 |
| 消息数量 | 一共导出了多少条聊天消息 |
| 保存位置 | 文档放在了哪里 |
| 索引编号 | 人才库里的记录编号 |

## 查看已同步的聊天

同步完成后，聊天记录不会自动展示全文（这是为了保护隐私）。如果你想查看：

> **你：** 展示张伟的微信聊天记录

> **你：** 查看候选人 ID 440 的聊天时间线

系统会提醒你内容可能包含敏感信息，确认后再展示。

聊天记录的格式类似这样：

```
## 2026-04-15 10:30 张伟
您好，我看到您发的职位信息挺感兴趣的

## 2026-04-15 10:32 顾问
您好！方便聊一下吗？这个岗位base在北京……

## 2026-04-15 10:35 张伟
可以的，我目前在北京，薪资范围大概是多少？
```

每条消息前面有时间戳和发送人，方便你快速定位到关键对话。

## 顺带更新候选人联系方式

同步聊天记录的时候，如果你还发现了候选人的新联系方式，可以一起更新：

> **你：** 同步候选人张伟的微信聊天记录，联系人是"张伟-阿里"，时间从 2026-04-01 到 2026-04-30，顺便更新他的邮箱为 zhangwei@alibaba.com

> **你：** 同步聊天的时候把李明的手机号也更新一下，13800138000

系统会先展示旧值和新值的对比：

```
联系方式更新确认：
- 邮箱：原值"无" → 新值"zhangwei@alibaba.com"
确认更新吗？
```

你说"确认"后，联系方式和聊天记录一起更新。

## 常见使用场景与示例

### 场景一：初次沟通后归档

你刚和候选人在微信上聊完，想把对话保存下来。

> **你：** 同步候选人王芳的微信聊天记录，联系人是"王芳-美团"，时间从 2026-05-01 到 2026-05-01

这里时间范围精确到某一天，只同步当天的聊天。

### 场景二：定期同步最近一个月

你习惯每月初同步上个月的聊天记录。

> **你：** 同步候选人赵磊的微信聊天，联系人是"赵磊"，时间从 2026-04-01 到 2026-04-30

### 场景三：群聊沟通记录

你在一个行业交流群里和候选人有过互动。

> **你：** 同步候选人陈刚在"上海互联网人脉群"的群聊记录，时间从 2026-03-15 到 2026-03-15

群聊同步的操作跟个人聊天完全一样，只是联系人的名字换成群名。

### 场景四：同步的同时更新联系方式

在聊天中候选人给了你他的邮箱和手机号。

> **你：** 同步候选人刘洋的微信聊天记录，联系人是"刘洋字节"，时间从 2026-05-10 到 2026-05-10，更新邮箱 liuyang@bytedance.com，手机 13900139000

### 场景五：同步后查看关键对话

同步完之后想快速回顾一下聊了什么。

> **你：** 展示候选人刘洋最近的微信聊天记录

系统会展示聊天全文，你可以快速定位到薪资讨论、面试安排等关键信息。

### 场景六：候选人在微信里用了备注名

你在微信里给候选人加了备注（比如"张伟-阿里-P8"），同步时用备注名就行。

> **你：** 同步候选人张伟的聊天记录，联系人是"张伟-阿里-P8"，时间从 2026-05-01 到 2026-05-07

## 聊天记录保存在哪里

同步后的聊天记录保存在你电脑上的文档文件里，每个文件都以"候选人编号-姓名-同步时间"命名，方便辨认。

| 保存内容 | 保存位置 | 说明 |
|----------|----------|------|
| 聊天全文 | 文档文件 | 完整的聊天内容，按时间排列 |
| 归档索引 | 人才库 | 记录"谁的聊天、什么时间范围、文件在哪"，不保存聊天内容本身 |

这种设计的好处是：
- 人才库里不会塞满聊天文字，保持干净
- 聊天文件是独立的，即使删除候选人，聊天记录文件也不会被自动删除

---

## 隐私与安全说明

| 规则 | 说明 |
|------|------|
| 必须指定时间范围 | 系统不会一键导出你和某人的所有聊天记录，你必须告诉它具体的时间段 |
| 不主动展示全文 | 同步完成后不会自动把聊天内容展示出来，需要你明确要求查看 |
| 聊天内容不入库 | 聊天文字只保存在文件里，不会写入人才库 |
| 手机号、微信号不索引 | 聊天中出现的手机号和微信号不会被系统自动提取和建立索引 |
| 删除候选人不删聊天 | 删除一个候选人时，他名下的聊天记录文件不会被自动删除，需要单独处理 |

## 常见问题

### 问：同步聊天记录时，微信需要打开吗？

是的。聊天记录是从微信里读取的，同步过程中微信需要在电脑上处于登录状态。

### 问：我能不能一次性同步所有候选人的聊天？

不行。每次同步必须指定一位候选人、一个微信联系人（或群）和明确的时间范围。这是为了保护隐私——你只能导出你确实需要的那部分聊天。

### 问：同一个候选人的聊天可以同步多次吗？

可以。比如你今天同步了4月的聊天，下个月又同步了5月的聊天，它们会各自保存为独立的文件。人才库里会分别记录这两次同步。

### 问：同步的聊天记录能修改吗？

聊天记录是只读的归档文件，不建议修改。如果你发现同步内容有误，可以重新同步一次。

### 问：群聊和个人聊天有什么区别？

操作方式完全一样。唯一区别是"联系人名字"那里，个人聊天填微信里那个人的名字，群聊填群的名字。

### 问：微信联系人的名字怎么确定？

就是你打开微信，在聊天列表或通讯录里看到的那个人（或群）的名字。如果你给对方加了备注，就用备注名。

### 问：时间格式怎么写？

以下写法都可以：

- `2026-05-01` — 某一天（从 0 点到 24 点）
- `2026-05-01 10:00` — 某天的某个时间点
- `2026-05-01 10:00:00` — 精确到秒

如果你只说"最近一个月"，系统会帮你算出具体的日期范围并跟你确认。

### 问：同步失败了怎么办？

常见的失败原因：

| 问题 | 原因 | 解决办法 |
|------|------|----------|
| 找不到联系人 | 微信里的名字和你告诉系统的不一致 | 确认微信里实际的联系人名字，重新同步 |
| 候选人不存在 | 人才库里没有这个人 | 先把候选人录入人才库，再同步聊天 |
| 0 条消息 | 这个时间范围内没有聊天记录 | 检查时间范围是否正确 |
| 导出工具不可用 | 微信导出功能未安装或未启动 | 联系技术支持 |

### 问：候选人删了之后，之前同步的聊天还在吗？

聊天记录文件还在。删除候选人只是删掉了人才库里的档案和索引，聊天文件不会自动删除。如果你确实想删掉聊天文件，需要单独处理。
