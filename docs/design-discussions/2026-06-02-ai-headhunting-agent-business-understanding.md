# AI 猎头公司 Agent 体系业务理解（2026-06-02）

## 背景

本讨论来自一组关于 AI 猎头公司 Agent 架构和业务蓝图的需求说明。用户希望表达的重点不是技术如何实现，而是 AI 对猎头业务的价值贡献：AI 如何帮助公司沉淀组织级资产，如何帮助顾问提高寻访、推荐、跟进和关系维护效率，以及公司与顾问之间如何通过数据和服务形成价值交换。

本文只整理纯业务理解，不展开后续产物形态或实现方案。

## 总体目标

目标是构建一个 AI 猎头公司的 Agent 体系。这个体系由公司端和个人端组成：

1. 公司端提供中心化的公司 Agent，承载组织级服务、数据资产、知识资产和协作连接。
2. 个人端提供每个顾问自己的个人 Agent，承载个人工作台、轻量级个人库、个人工具和个人知识沉淀。
3. 公司 Agent 与个人 Agent 之间形成双向价值交换：个人从公司获得 AI 服务、数据、信息和洞察；公司从个人获得数据贡献、质量反馈、业务经验和知识沉淀。

系统的业务核心不是让 AI 替代顾问，而是让 AI 扩大顾问能力，降低低价值事务成本，提升岗位理解、寻访决策、线索复用、精准推荐、深度跟进和长期关系维护能力。

## 业务参与方

### 公司

公司负责建设和运营组织级 Agent 能力，沉淀可复用资产，并制定价值交换、数据隔离、贡献奖励和机会归属规则。

公司获得的核心价值包括：

1. 公司人才库持续增长。
2. 公司知识库持续增长。
3. 线索池和联系方式资产持续积累。
4. 岗位画像、匹配评分和推荐策略不断被反馈校准。
5. 顾问个人经验逐步沉淀为组织能力。

### 顾问

顾问通过个人 Agent 使用公司 Agent 的服务，也通过日常业务持续向公司贡献数据、反馈和经验。

顾问获得的核心价值包括：

1. 更快理解岗位和制定寻访计划。
2. 使用公司级多渠道寻访服务获得人才线索。
3. 使用公司人才库和线索池补齐联系方式。
4. 获得行业、企业和人选关联洞察。
5. 获得深度跟进 SOP 和自动化触达支持。
6. 通过贡献数据和知识获得积分或酬劳。

### 人选

人选是被寻访、评估、推荐和长期维护关系的对象。系统不只在有职位时触达人选，也支持在合适时机进行回访、关系维护和情绪价值提供。

### 客户与岗位

客户和岗位是业务需求来源。公司 Agent 需要帮助顾问理解岗位需求，形成岗位画像、推荐人才画像和寻访策略，并持续通过反馈优化岗位画像模板。

## 公司 Agent 的业务能力

公司 Agent 是中心化的业务能力中枢，应包含以下组织级服务。

### 渠道寻访

公司 Agent 提供多渠道寻访服务，帮助顾问获取人才线索。顾问可以调用该服务进行寻访，而不是每个人完全独立重复建设渠道能力。

### 线索池管理

公司 Agent 负责管理线索池。线索池沉淀来自公司服务、顾问贡献和多渠道寻访产生的人才线索，并支持后续跟进、复用和交易。

### 公司人才库管理

公司人才库是组织级人才资产。顾问在业务过程中获得的人选信息、联系方式、跟进状态和关键备注，可以按照统一规则同步到公司人才库。

公司人才库既是推荐和寻访的基础，也是联系方式交易、线索复用和业务洞察挖掘的核心数据资产。

### 公司知识库管理

公司知识库沉淀组织级 know-how，包括岗位理解、行业知识、寻访策略、推荐经验、沟通经验、优秀案例、失败案例和顾问个人总结。

个人 Agent 中产生的 know-how 和知识总结，应尽量及时沉淀到公司知识库里。

### 组织协作空间连接与管理

公司 Agent 应连接和管理组织协作空间，例如飞书。飞书可以作为业务协作、信息同步、任务沟通、知识沉淀和交付协同的组织入口。

### 行业与企业动态获取

公司 Agent 应持续获取外部信息，包括：

1. 行业资讯。
2. 目标企业动态。
3. 相关领域前沿进展。

这些信息需要与公司人才库、线索池和本地人选信息进行关联挖掘，形成对顾问有价值的业务洞察。

## 个人 Agent 的业务能力

个人 Agent 是顾问个人工作台，应包含以下能力。

### 轻量级个人人才库

个人 Agent 应包含轻量级个人人才库，用于管理顾问自己正在跟进、维护和判断的人选。

个人人才库承载顾问的日常工作过程，但不应成为孤立资产。按统一规则，关键数据应同步到公司人才库。

### 个人定制化工具包

个人 Agent 应包含顾问定制化工具包，用于支持顾问自己的工作习惯和业务流程，例如岗位理解、寻访计划、沟通准备、跟进提醒、备注整理等。

### 个人 know-how 和知识总结

顾问个人的业务经验、判断方法、沟通技巧、行业理解和失败经验，应先在个人 Agent 中沉淀，再及时同步到公司知识库，形成组织级复用能力。

## 核心业务场景

### 场景一：岗位咨询与寻访计划制定

顾问向公司 Agent 进行岗位咨询。

公司 Agent 输出：

1. 岗位画像。
2. 推荐人才画像。
3. 人才寻访策略。
4. 其他用于决策的岗位信息。

这些输出帮助顾问制定寻访计划，提高岗位理解和寻访方向判断的质量。

### 场景二：多渠道寻访与线索跟进

顾问调用公司 Agent 的寻访服务，进行多渠道寻访以获得人才线索。

后续流程包括：

1. 顾问获得候选人线索。
2. 顾问对线索进行跟进。
3. 顾问将获得的联系方式更新到本地个人人才库。
4. 顾问将跟进日志和备注更新到本地个人人才库。
5. 相关信息按照统一规则同步到公司人才库。

该场景的业务价值是让公司级寻访服务和个人跟进动作形成闭环，既提升个人效率，也沉淀公司资产。

### 场景三：推荐质量反馈与精准推荐优化

顾问会对公司 Agent 寻访和推荐的人选进行质量反馈。

反馈数据进入公司 Agent 后，形成反向优化循环：

1. 顾问反馈推荐人选质量。
2. 公司 Agent 根据反馈重新评估推荐结果。
3. 系统不断完善人选匹配评分模型。
4. 系统不断完善岗位画像模板。
5. 推荐精准度逐步提升。

该场景强调的是业务学习闭环：推荐不是一次性输出，而是通过顾问反馈持续改进。

### 场景四：联系人线索价值交换

顾问可以将已经获得联系方式的人选信息同步给公司人才库，并获得贡献积分。

顾问也可以使用代币向公司人才库请求自己当前正在跟进线索的联系方式。如果公司人才库中已有该线索的联系方式，则可以进行交易。

该场景的关键规则包括：

1. 联系方式交易不排他。
2. 一个顾问可以用代币购买某个人选联系方式。
3. 其他顾问也可以购买同一个人选联系方式。
4. 联系方式不是独占资产。
5. 有效联系方式的贡献奖励只给第一个贡献者。
6. 后续重复贡献同一联系方式，不再获得同等奖励。

该场景的业务价值是把个人掌握的联系方式转化为公司资产，同时让顾问能通过贡献获得可消费权益。

### 场景五：行业洞察与人选深度跟进

公司 Agent 持续获取行业资讯、目标企业动态和相关领域前沿进展。

公司 Agent 将这些信息与本地人选信息进行关联挖掘，形成有价值的关联洞察，并推送给顾问。

顾问可以基于这些洞察进行人选深度跟进，例如：

1. 结合目标企业变化判断人选流动意愿。
2. 结合行业变化判断人选机会窗口。
3. 结合候选人经历和外部动态设计更有价值的沟通内容。
4. 在没有具体职位时，也能进行有内容、有温度的关系维护。

### 场景六：深度跟进 SOP 与自动化触达

系统应提供完善的人选深度跟进 SOP 和自动化触达网络，帮助顾问持续维护人选关系。

该场景强调：

1. 不只是有职位时才联系人选。
2. 应在适当时机进行人选回访。
3. 应持续提供情绪价值。
4. 应持续维护深度关系。
5. 应通过 SOP 和自动化触达降低顾问维护长期关系的成本。

## 价值交换机制

整个体系遵循价值交换原则。公司端和个人端之间，不应是单向索取，而应形成可计量、可兑换、可激励的双向机制。

### 代币

代币是服务的有偿支付单位。

代币可以用于：

1. 使用公司 Agent 的服务。
2. 获取公司数据。
3. 获取公司信息。
4. 请求公司人才库中的联系方式。
5. 消费公司 Agent 产生的洞察、推荐或其他增值能力。

代币未来可以演进为个人账户和支付体系。

### 贡献积分

贡献积分是数据价值度量单位。

贡献积分用于衡量顾问对公司的贡献，例如：

1. 贡献有效联系方式。
2. 贡献跟进数据。
3. 贡献质量反馈。
4. 贡献个人 know-how。
5. 贡献知识总结。
6. 贡献可复用业务经验。

贡献积分不是直接支付工具，而是衡量贡献价值的业务指标。

### 积分转代币

系统应建立贡献积分转化为代币的机制。

该机制用于实现价值交换：

1. 顾问向公司贡献数据或知识，获得贡献积分。
2. 贡献积分按规则转换为代币。
3. 顾问使用代币消费公司 Agent 服务或公司数据。

通过该机制，个人贡献可以转化为个人可消费权益。

### 公司对个人贡献的有偿回报

公司从个人获得贡献时，也需要提供有偿酬劳或等价激励。

这些贡献包括但不限于：

1. 联系方式。
2. 有效线索。
3. 跟进日志。
4. 推荐质量反馈。
5. 岗位理解经验。
6. 寻访经验。
7. 人选关系维护经验。

## 数据同步与数据隔离

### 统一同步规则

个人 Agent 和公司 Agent 之间应遵循统一的数据同步规则。

个人库是顾问的本地工作台，公司人才库是最终沉淀的组织级资产。顾问在工作中获得的人选联系方式、跟进日志、备注和状态，应按照统一规则同步到公司人才库。

### 顾问间数据隔离

系统应保证顾问与顾问之间的信息和数据隔离。

数据隔离意味着：

1. 顾问不能直接看到其他顾问的完整个人工作过程。
2. 顾问不能直接占有其他顾问的跟进关系。
3. 公司人才库可以沉淀组织级数据，但顾问之间仍应保持边界。
4. 数据共享应通过公司 Agent、人才库规则、积分和代币机制完成。

## 机会归属与竞争规则

### 同一人选允许多人跟进

同一个人选允许不同顾问同时进行跟进。

这是为了避免过早锁死资源，也允许不同顾问基于不同客户、不同岗位和不同关系路径进行良性竞争。

### 先到先得的适用范围

先到先得主要适用于：

1. 成单归属。
2. 推荐机会归属。

谁先推动到有效成单，收益和归属优先。谁先完成对某个 JD 或客户机会的人选推荐，推荐机会优先。

### 联系方式交易不排他

联系方式交易不适用排他规则。

一个顾问可以通过代币获得某个人选联系方式，其他顾问也可以通过代币获得同一个联系方式。

但联系方式贡献奖励只给第一个贡献者。

### 良性竞争

系统应鼓励顾问之间良性竞争：

1. 不因一个顾问先接触某人选就永久锁定该人选。
2. 不让顾问直接抢夺彼此的私有工作过程。
3. 通过成单归属、推荐机会归属和贡献奖励规则明确收益边界。
4. 通过数据隔离和价值交换机制降低内部冲突。

## AI 的业务价值贡献

AI 在该体系中的价值贡献包括：

1. 帮助顾问理解岗位，形成岗位画像和推荐人才画像。
2. 帮助顾问制定寻访计划。
3. 提供多渠道寻访服务，提升线索获取效率。
4. 帮助公司管理线索池和人才库。
5. 帮助公司管理知识库，并把个人经验沉淀为组织知识。
6. 基于顾问反馈优化推荐质量。
7. 基于反馈持续完善人选匹配评分模型。
8. 基于反馈持续完善岗位画像模板。
9. 将行业资讯、目标企业动态和领域前沿进展与人选信息关联，生成业务洞察。
10. 将洞察推送给顾问，支持深度跟进。
11. 提供深度跟进 SOP 和自动化触达网络。
12. 支持不依赖具体职位的长期关系维护。
13. 支持公司和个人之间的数据价值交换。
14. 支持顾问之间数据隔离下的良性竞争。

## 业务闭环总结

该体系可以概括为五个闭环：

1. 岗位理解闭环：岗位咨询、岗位画像、寻访策略、推荐计划。
2. 寻访交付闭环：公司 Agent 寻访、顾问跟进、个人库记录、公司库同步。
3. 推荐优化闭环：推荐人选、顾问反馈、评分模型优化、画像模板优化、精准推荐提升。
4. 价值交换闭环：个人贡献、贡献积分、积分转代币、代币消费公司服务和数据。
5. 关系维护闭环：外部动态、关联洞察、深度跟进、SOP 触达、长期人选关系维护。

最终目标是把猎头顾问的个人经验、线索、人脉和判断，逐步转化为公司级可复用资产；同时让公司级 AI、数据和知识能力反向赋能个人顾问，形成可持续增长的 AI 猎头公司业务系统。
