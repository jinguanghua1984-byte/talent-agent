# 脉脉 AI Infra 人才搜索执行计划

## 目标

在脉脉人才搜索页中，围绕 AI Infra、大模型训练、推理、分布式训练、训练/推理框架、算子、异构计算、智算平台等方向，手动搜索并沉淀高匹配候选人。

本计划采用二八法则：优先把 80% 时间投入最可能产出有效候选人的公司、岗位名和关键词组合，减少泛搜索带来的噪音。

## 搜索原则

1. 目标公司必须命中，不做泛行业海捞。
2. 职位名称必须逐个填写，不把多个岗位名堆在同一个职位字段里。
3. 泛岗位只在技术关键词同时命中时查看，例如“算法工程师”必须同时出现大模型、训练、推理、框架、算子、异构、智算、Token 调度或加速等关键词。
4. 训练师、运营、HR、人事、招聘、销售、数据标注、审核等岗位直接跳过。
5. 学历优先级为 C9、985、211、本科；大专候选人不进入主筛选池。
6. 不建议在搜索页用年龄做硬筛，执行上优先用工作年限、职级、项目复杂度和技术深度判断候选人层级。

## 搜索页字段填写规则

| 字段 | 第一轮填写方式 | 说明 |
| --- | --- | --- |
| 城市地区 | 不填 | 结果过多时再按北京、上海、杭州、深圳筛选 |
| 学历要求 | 本科及以上 | 后续人工优先看 C9、985、211 |
| 工作年限 | 2-10 年 | 主搜索池 |
| 就职公司 | 必填 | 一次填一个公司或一个公司主体 |
| 职位名称 | 必填 | 一次只填一个职位名称 |
| 行业方向 | 可不填 | 避免平台分类不准导致漏人 |
| 毕业学校 | 第一轮不填 | 二次筛选时再用 |
| 专业 | 第一轮不填 | 二次筛选时再看计算机、软件、自动化、电子、数学等 |
| 年龄 | 不填 | 用工作年限和经历质量替代 |
| 顶部关键词 | 填产品词或技术词 | 优先选择“任意关键词”匹配 |

## 候选人分层

### A 档

满足以下多数条件：

- 第一梯队或高密度第二梯队公司。
- 职位名称精准命中 AI Infra、大模型训练、推理、分布式、框架、算子、异构、智算等方向。
- 简历包含 2 个以上核心技术关键词。
- C9、985、211 或有强开源/强项目经历。

### B 档

满足以下多数条件：

- 目标公司命中。
- 岗位是算法、机器学习、深度学习、平台研发、系统研发等偏泛职位。
- 简历中有明确训练、推理、框架、算子、GPU、智算、分布式系统等经历。
- 学历、年限或岗位标题有一项略弱，但技术经历可补足。

### C 档

满足以下条件之一：

- 公司符合，但岗位偏业务算法、推荐、搜索、广告，Infra 证据不足。
- 简历技术词较少，需要后续人工确认。
- 只作为补充池，不优先沟通。

### 直接淘汰

- 目标公司不匹配。
- 职位名明显不相关。
- 简历没有任何 AI Infra、机器学习、深度学习、训练框架、推理框架、RL、SFT、算子、异构、微调、Token 调度平台、智算、加速相关信息。
- 职位为训练师、运营、HR、人事、招聘、销售、数据标注、审核等。
- 学历为大专且没有极强技术证据。

## 每日执行节奏

假设每天投入 2 小时。

| 日期 | 搜索任务 | 目标产出 |
| --- | --- | --- |
| Day 1 | 字节、阿里、百度、快手；只搜第一批职位名称 | 每家公司沉淀 10-15 个 A/B 档 |
| Day 2 | 第一梯队继续搜第二批职位名称，再搜泛岗位 | 补足第一梯队候选池 |
| Day 3 | 月之暗面、DeepSeek、MiniMax、智谱、阶跃星辰、生数科技、爱诗科技 | 每家公司沉淀 5-10 个 A/B 档 |
| Day 4 | 华为、腾讯、蚂蚁金服、商汤、硅基流动、一流科技、上海人工智能实验室 | 补强 Infra、智算、推理方向 |
| Day 5 | 第三梯队技术专向补扫 | 每家公司沉淀 3-5 个技术候选人 |
| Day 6 | 复盘 A/B 档候选人，补搜缺口公司和缺口岗位 | 形成第一批可沟通名单 |

## 停止规则

1. 每个搜索条件最多查看前 3 页。
2. 连续 2 页没有 A/B 档候选人，停止该搜索条件。
3. 泛岗位搜索中，如果前 1 页多数候选人没有训练、推理、框架、算子、异构、智算等证据，停止该职位名称。
4. 第四、五梯队只在第一、二、三梯队候选池不足时补扫。

## 人工复核清单

查看候选人详情时，按以下顺序判断：

1. 当前或最近一段经历是否在目标公司。
2. 职位名称是否接近 AI Infra、大模型训练、推理、分布式、框架、算子、异构、智算方向。
3. 简历内容是否命中核心技术词。
4. 项目是否偏底层工程，而不是纯业务策略、运营配置或数据标注。
5. 学历是否为 C9、985、211、本科及以上。
6. 工作年限是否适合当前岗位层级。
7. 是否有开源框架、GPU 集群、大模型训练、推理服务、模型部署、加速优化等硬证据。

## 最小可执行流程

每天执行时只需要按这个顺序操作：

1. 选择当天公司清单。
2. 就职公司填一个公司。
3. 职位名称按第一批、第二批、第三批逐个填写。
4. 顶部关键词先用产品词，再用 K1-K5 技术关键词包轮换。
5. 每个搜索条件最多看 3 页。
6. 只记录 A/B 档候选人。
7. 当天结束后复盘哪些公司和岗位产出最高，第二天优先复用高产组合。
