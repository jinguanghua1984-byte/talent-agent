# public-search Skill 改善设计方案

> 基于 2026-04-10 全链路测试讨论，将 public-search 从「单次搜索工具」改造为「策略驱动的协作搜索系统」。

## 1. 现状与问题

### 当前流程
```
/public-search <关键词> → AI 内部生成策略 → 搜索 → 输出候选人 → 结束
```

### 核心问题

| # | 问题 | 影响 |
|---|------|------|
| P1 | 策略是 AI 内部步骤，用户无法参与 | 搜索方向可能偏离用户意图 |
| P2 | 无引导入口，没有 JD 也没有策略时盲搜 | 搜索质量低 |
| P3 | 一次性执行，无复盘迭代 | 同样的错误反复犯 |
| P4 | 渠道/关键词不区分岗位类型 | 产品经理搜 GitHub 无意义 |
| P5 | 搜索经验不积累 | 每次 from scratch |

## 2. 目标设计

### 新流程
```
/public-search → 引导入口 → 协作策略 → 执行搜索 → 反馈归因 → 迭代优化 → 策略沉淀
                              ↑___________|                                        |
                              |_________迭代循环__________________________________|
```

### 设计原则

1. **策略先行**：没有确认的策略不执行搜索
2. **人机协作**：策略是用户和 AI 共同编辑的产物
3. **效果可衡量**：每个策略元素的效果可归因
4. **经验可复用**：搜索策略三层沉淀（Instance → Template → Universal）
5. **岗位感知**：根据岗位类型自动调整渠道和评估维度

## 3. 详细设计

### 3.1 引导式启动

```
/public-search
```

AI 检测输入类型，引导进入对应路径：

```
📋 你想搜什么人？

1. 我有 JD → 请给出 JD 文本或 JD ID
2. 我有搜索思路 → 请描述你的搜索策略
3. 帮我想想 → 请告诉我目标岗位和核心要求
```

| 用户输入 | 入口 | 后续 |
|---------|------|------|
| JD 文本 / JD ID | JD 驱动 | AI 生成策略 → 用户审查 → 确认 |
| 搜索策略描述 | 策略驱动 | AI 审查 → 优化建议 → 用户确认 |
| 岗位描述 / 什么都不给 | 引导模式 | AI 引导补充信息 → 生成策略 |

### 3.2 协作策略生成

#### 3.2.1 策略文件格式

策略保存为「活文档」，搜索过程中持续更新：

```markdown
# 搜索策略：{JD标题}

## 元信息
- JD ID: {id}
- 创建时间: {timestamp}
- 当前轮次: {N}
- 来源模板: {template-id 或 "无"}

## 目标画像
（描述目标候选人的核心特征，1-3 种画像）

## 搜索策略
### 关键词矩阵
| # | 关键词 | 渠道 | 优先级 | 状态 |
|---|--------|------|--------|------|
| 1 | "AI Agent" "产品经理" | Google | 高 | active |
| 2 | "百万DAU" "产品经理" | Google | 高 | removed(轮次2: 命中率0%) |

### 渠道计划
| 渠道 | 搜索条数 | 权重 |
|------|---------|------|
| LinkedIn | 3 | 高 |

## 执行记录
### 轮次 1 ({timestamp})
#### 归因表
| 策略元素 | 搜索量 | 候选人 | 命中率 |
|---------|--------|--------|--------|
#### 覆盖缺口
| JD 要求 | 状态 |
|---------|------|
#### AI 反思
（本轮学到了什么）

### 轮次 2 ({timestamp})
...
```

#### 3.2.2 JD 驱动路径

```
Step 1: AI 读取 JD → 提取关键信息
Step 2: 匹配 Template（如有）→ 获取基础策略
Step 3: 应用 Universal 规则 → 过滤无效做法
Step 4: 生成初版策略 → 展示给用户
Step 5: 用户审查 → 补充/修改
Step 6: 合并为最终策略 → 用户确认 → 执行
```

#### 3.2.3 策略驱动路径

```
Step 1: 用户给出搜索策略（文本/结构化）
Step 2: AI 解析策略 → 结构化展示
Step 3: AI 审查 → 给出优化建议（渠道、关键词、画像）
Step 4: 用户采纳/拒绝 → 定稿
Step 5: 确认 → 执行
```

### 3.3 执行搜索

执行逻辑与现有 Skill 基本一致，关键改动：

1. **按策略元素逐条执行**，而非一次性全搜
2. **每条策略元素独立记录结果**，为归因表提供数据
3. **候选人与策略元素关联**，记录每个候选人来自哪个搜索

### 3.4 反馈归因

搜索完成后，自动生成三层反馈：

#### 层 1：归因表

```
📊 搜索效果归因

| # | 策略元素 | 搜索量 | 候选人 | 噪音 | 命中率 | 状态 |
|---|---------|--------|--------|------|--------|------|
| 1 | "AI Agent" "产品经理" Google | 23 | 3 | 20 | 13% | ✅ 保留 |
| 2 | "百万DAU" "产品经理" Google | 15 | 0 | 15 | 0% | ❌ 建议移除 |
| 3 | "豆包" "产品经理" LinkedIn | 8 | 2 | 6 | 25% | ✅ 保留 |
```

#### 层 2：覆盖缺口

```
📋 JD 覆盖分析

| JD 要求 | 已匹配 | 缺口 |
|---------|--------|------|
| 5-10年产品经验 | ✅ 5人 | — |
| 百万DAU+ | ❌ 0人 | 公开渠道无法验证 |
| AI产品实战 | ⚠️ 2人 | 深度不确定 |
```

#### 层 3：AI 反思 + 建议

```
🤖 本轮反思

• 渠道：LinkedIn(25%) > Google(13%)，产品经理岗位 LinkedIn 更有效
• 关键词：带公司名(豆包)比泛词(AI产品经理)更精准
• 缺口：百万DAU 在公开渠道不可验证

💡 优化建议：
A. 关键词：去掉"百万DAU"，加"ChatGPT Notion AI 产品经理"
B. 渠道：提高 LinkedIn 优先级
C. 画像：收窄到大厂AI部门PM

你的想法？
```

### 3.5 迭代循环

用户基于反馈决定下一步：

```
用户选择：
  → 采纳某些建议 → 更新策略 → 再搜一轮
  → 提出自己的优化 → AI 审查 → 更新策略 → 再搜一轮
  → 满意当前结果 → 结束搜索 → 进入后续流程（screen/report）
  → 放弃 → 记录失败原因 → 沉淀到策略库
```

每轮迭代：
1. 更新策略文件中的「执行记录」
2. 更新策略元素的状态（active/removed/modified）
3. 重新计算归因表（累计）
4. 重新分析覆盖缺口

### 3.6 策略沉淀

#### Instance → Template 提升条件

当同一岗位类型积累 3+ 次 Instance 搜索时，AI 提议：

```
💡 发现 3 次「AI产品经理」搜索的共性：
• LinkedIn 命中率稳定在 20%+
• GitHub 命中率接近 0%
• 带具体产品名的关键词效果最好

是否创建「AI产品经理」岗位模板？
```

用户确认后创建 Template，后续同类搜索自动推荐。

#### Template → Universal 提升条件

当多个 Template 出现相同规律时（如「具体公司名 > 泛词」出现在 3+ 个 Template），AI 提议提升为 Universal 规则。

### 3.7 岗位类型感知

#### 渠道推荐（基于岗位类型）

| 岗位类型 | 首选渠道 | 辅助渠道 | 排除渠道 |
|---------|---------|---------|---------|
| 技术开发 | GitHub, Google | 技术社区, LinkedIn | 行业媒体 |
| 产品经理 | LinkedIn, Google | 行业媒体, 即刻 | GitHub, Scholar |
| 设计师 | Dribbble, Behance | LinkedIn, 小红书 | GitHub, Scholar |
| 管理/高管 | LinkedIn, 行业媒体 | Google | GitHub |
| 学术/研究 | Scholar, 知网 | Google, GitHub | LinkedIn |

#### 评估维度（供 screen 使用）

| 岗位类型 | 核心维度 | 权重调整 |
|---------|---------|---------|
| 技术岗 | 岗位匹配/技能覆盖/经验深度/行业背景/稳定性 | 默认 30/25/20/15/10 |
| 产品岗 | 产品思维/C端经验/AI理解力/行业洞察/执行力 | 30/25/20/15/10 |
| 设计岗 | 设计能力/作品集/审美/行业经验/协作力 | 30/25/20/15/10 |

## 4. 数据结构

### 4.1 目录结构

```
data/
├── search-strategies/
│   ├── instances/           # 单次搜索策略
│   │   └── {date}-{slug}.md
│   ├── templates/           # 岗位模板
│   │   └── {role-type}.md
│   └── universal-rules.json # 通用规则
├── jds/                     # JD 数据（不变）
├── candidates/              # 候选人数据（不变）
└── output/                  # 搜索输出（不变）
```

### 4.2 Template 文件格式

```markdown
# 岗位模板：AI产品经理

## 适用条件
- 职位关键词包含：产品经理、PM、Product Manager
- 行业关键词包含：AI、大模型、Agent、LLM

## 推荐渠道
1. LinkedIn（权重：高）
2. Google（权重：高）
3. 行业媒体 / 即刻（权重：中）

## 推荐关键词模式
- 带具体产品名 > 泛词
- 带 AI 公司名 > 不带
- 避免使用不可验证的指标（百万DAU等）

## 排除渠道
- GitHub（命中率 < 2%）
- Google Scholar（非学术岗）

## 统计数据
- 基于搜索次数：5
- 平均命中率：18%
- 最后更新：2026-04-10
```

### 4.3 Universal Rules 格式

```json
{
  "rules": [
    {
      "id": "universal-001",
      "rule": "具体公司名/产品名作为关键词比泛词命中率更高",
      "based_on": ["template-ai-pm", "template-backend", "template-data"],
      "confidence": "high",
      "created_at": "2026-04-10"
    },
    {
      "id": "universal-002",
      "rule": "不可量化指标（百万DAU、营收规模）在公开渠道无法验证，改搜具体产品名",
      "based_on": ["template-ai-pm", "template-frontend"],
      "confidence": "high",
      "created_at": "2026-04-10"
    }
  ]
}
```

## 5. 实施计划

### Phase 1：核心流程改造
- [ ] 重写 SKILL.md：引导入口 + 协作策略 + 反馈归因
- [ ] 更新 search-sources.md：增加岗位类型渠道推荐表
- [ ] 新增策略文件模板
- [ ] 新增 Instance 策略存储目录

### Phase 2：迭代与沉淀
- [ ] 实现归因表自动生成
- [ ] 实现覆盖缺口分析
- [ ] 实现 AI 反思和建议生成
- [ ] 新增 Template 创建流程

### Phase 3：智能化
- [ ] 实现 Template 自动匹配
- [ ] 实现 Universal 规则提炼
- [ ] 实现 Template → Universal 提升流程
- [ ] screen 评估维度按岗位类型分化

### MVP 范围建议

先做 Phase 1，验证「协作策略 + 反馈归因」的用户体验。Phase 2 和 3 在积累足够搜索数据后逐步推进。

## 6. 与现有 Skill 的关系

| Skill | 关系 | 需要改动 |
|-------|------|---------|
| public-search | 本次重设计主体 | ✅ 大改 |
| platform-match | 搜索后信息丰富，不受影响 | ❌ |
| screen | 接收候选人，评估维度可能需适配 | ⚠️ 可选：岗位类型感知 |
| report | 不受影响 | ❌ |
| data-manager | 需要新增策略相关的数据管理命令 | ⚠️ 可选：后期 |
