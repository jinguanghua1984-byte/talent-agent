## Part 1：业务要点讨论

### Slide 3：Part 1 总览：业务重构的三个核心议题

核心信息：

1. 人才数据库：废旧立新，重起炉灶。
2. Sourcing：SOP 自动化，解放顾问。
3. 人选深度运营：目标人选层级上升的基础。

讲述重点：

1. 三个议题分别对应数据底座、获客/寻访效率、人选长期关系能力。
2. AI 猎头公司的真正基础不是模型本身，而是业务数据、流程和人选关系的重构。

### Slide 4：人才数据库：废旧立新，重起炉灶

核心观点：

现有人才数据库应废旧立新，重起炉灶。

原因：

1. 旧数据库是外采产品，无法改造，不可控。
2. 架构老旧，不适配 AI 能力。
3. 业务重心迁移，既有人选数据质量存疑。

讲述重点：

1. 旧数据库的问题不是简单的数据量不足，而是不可控、不可改造、无法适配未来 AI 能力。
2. 如果数据底座不重构，后续 AI 能力只能停留在外围工具层。

### Slide 5：人才数据库建设关键：双轨过渡与新业务数据源

核心信息：

1. 既有数据库仍是管理和财务数据的主要来源，短期无法替代。
2. 既有数据库保留其订单、财务、管理的唯一数据来源定位，长期逐步替代。
3. 新数据库围绕业务提效和 AI 适配双核心定位建设。
4. 新数据库优先人才核心信息，逐步扩展业务过程数据。
5. 新数据库未来成为业务运营数据的唯一数据源。
6. 建立公司人才库和个人人才库两级架构。
7. 制定合适的权限策略和同步机制。
8. 解决公司-个人数据脱节、顾问间人选归属冲突等问题。

讲述重点：

1. 短期不是推翻全部旧系统，而是保留其管理和财务定位。
2. 新系统先服务业务提效和 AI 适配，再逐步成为业务运营主数据源。
3. 两级人才库是未来公司 Agent 和个人 Agent 协作的基础。

### Slide 6：人才数据库冷启动：择优导入、低信线索池与拓展汇总

核心信息：

1. 既有数据库择优导入。
2. 从脉脉等三方渠道接入低信线索池。
3. 低信线索池包括无联系方式、求职意向不明、弱关系等线索。
4. 汇总拓展需求，由公司统一拓展。
5. 或提供工具能力赋能顾问自行拓展，再统一汇总。

讲述重点：

1. 冷启动不是一次性迁移全部数据，而是择优导入。
2. 低信线索池先扩大覆盖，再通过后续跟进和数据补齐逐步提升质量。
3. 公司统一拓展和顾问工具化拓展可以并行。

### Slide 7：Sourcing：SOP 自动化，解放顾问

核心观点：

Sourcing 应通过 SOP 自动化解放顾问。

原因：

1. 价值 / 时间产出比太低。
2. 统一 SOP 是业绩保障。
3. 统一 SOP 也有利于引入 AI，形成规模效应。

讲述重点：

1. Sourcing 是高消耗环节，不能长期依赖顾问手工重复劳动。
2. 统一 SOP 不是为了僵化流程，而是为 AI 自动化和规模化提供基础。

### Slide 8：Sourcing 建设关键：中心化、个性化、成本与利益分配

核心信息：

1. 公司中心化与顾问个性化需要同时存在。
2. 需要区分能力的中心化与个性化。
3. 需要区分数据的中心化与个性化。
4. 需要评估 token/API 成本与人工成本。
5. 需要评估成本与时间收益。
6. 需要设计利益分配机制：
   - 公司 - 顾问
   - 顾问 - 顾问
   - 公司 - 服务商
7. 需要建立数据闭环，持续优化。
8. 数据闭环路径：推荐 -> 反馈 -> 评分模型微调 -> 二次推荐自评 -> 模型更新或保持不变。

讲述重点：

1. Sourcing 自动化不能只算技术可行性，还要算成本、时间和利益分配。
2. 推荐反馈闭环是长期优化的关键。
3. 中心化能力与顾问个性化能力之间需要明确边界。

### Slide 9：Sourcing 渠道现状：脉脉、Boss 与公开信息源

核心信息：

脉脉：

1. 列表和详情可以在监督下批量获取。
2. 每个账号约 500 次搜索/天。
3. 每 150 次会弹出验证码，目前需要人工干预，未来可以尝试自动化验证。

Boss：

1. 可以自动化操作 App，实现人选筛选、档案信息抓取、自动沟通、消息自动处理等场景。
2. 风控规则暂不清楚。
3. 有触达次数限制，且触达转化不佳。

GitHub / 搜索引擎 / 论文期刊：

1. 可以对指定产品团队、组织进行 list 抓取。
2. 有效信息有限，需要去其他渠道二次匹配，然后触达。

讲述重点：

1. 不同渠道的自动化价值不同，限制也不同。
2. 脉脉和 Boss 更接近招聘场景，但有风控和验证码约束。
3. 公开信息源适合扩展名单，但通常需要二次匹配和触达补齐。

### Slide 10：LinkedIn 路径：数据聚合 API 与 RPA 的取舍

核心信息：

方案 A：使用第三方数据聚合商。

1. 代表服务商包括 Proxycurl、Coresignal、Nubela、Bright Data。
2. Proxycurl 专门针对领英数据解析。
3. 优势是按次调用计费，对开发者友好。
4. 单次查询价格示例约 0.01 - 0.05 美元不等。
5. 支持直接输入领英 URL 返回结构化 JSON 简历数据。
6. 适合投喂给大模型做匹配分析。

方案 B：RPA / 浏览器自动化。

1. 优势是成本主要在服务器和研发上。
2. 劣势是领英风控极严。
3. 高频搜索和页面停留时间检测会带来账号封禁风险。
4. 需要极强的反爬对抗基础设施。

讲述重点：

1. LinkedIn 更适合分阶段处理，不适合无差别高频抓取。
2. 数据聚合 API 更适合深度匹配阶段。
3. RPA 成本低但风控风险高。

### Slide 11：LinkedIn 实战落地策略与增量渠道

核心信息：

LinkedIn 实战落地策略：

1. 寻访阶段：通过 RPA 或更廉价的搜索接口获取候选人基础名单和领英 URL。
2. 低价接口示例：2026 年新出的 Netrows，约 `€49 / 1 万次积分`。
3. 深度匹配阶段：针对初筛 URL 列表，通过 Proxycurl API 拉取深度结构化简历 JSON 数据。
4. 分析决策：将 JSON 连同 JD 一并提交给大模型，输出结构化匹配报告。

其他增量渠道：

1. 开源社区，例如 Hugging Face、GitHub 等。
2. 追踪顶级学术会议，例如 NeurIPS、CVPR、ACL 等的华人一作或通讯作者。
3. 混入高质量的认知圈层。

讲述重点：

1. 寻访阶段重在便宜、广覆盖、拿到名单和 URL。
2. 深度匹配阶段再使用高质量 API 和大模型。
3. 增量渠道的价值在于找到传统招聘平台覆盖不到的人。

### Slide 12：人选深度运营：目标人选层级上升的基础

核心观点：

人选深度运营能力是目标人选层级上升的基础。

核心问题：

如何实现前期积累和持续运营。

讲述重点：

1. 越高层级、越稀缺的人选，越不能只靠职位触达。
2. 深度运营能力决定能否长期建立信任。
3. AI 的价值在于降低持续运营成本，让长期关系维护可行。

### Slide 13：人选深度运营建设关键：行业洞察与独特价值

核心信息：

快速构建行业洞察：

1. 行业知识库。
2. 行业与人才图谱。
3. 行业词典。
4. 咨询洞察。

为人选提供独特价值：

1. 知识交换，例如商业化案例、公司政治、谈薪策略、行业周期洞察等。
2. 作为人脉连接的节点。
3. 激活联系人 / 朋友圈里的人脉。
4. AI 可以有效降低运营成本，让持续运营可行。

讲述重点：

1. 人选深度运营不是简单群发消息，而是持续提供有价值的信息和连接。
2. 公司需要积累行业知识和人才图谱，才能让顾问持续提供独特价值。

### Slide 14：自动化触达网络：用洞察驱动长期关系维护

核心信息：

自动化触达网络包括：

1. 信息抓取与匹配工作流。
2. 实时追踪重点候选人所在赛道的融资动态。
3. 实时追踪技术突破。
4. 实时追踪行业动荡。
5. 在关键节点带着洞察去关心对方。
6. 为人选提供情绪价值和客观建议。

讲述重点：

1. 自动化触达不是冷冰冰的批量外呼，而是基于洞察的长期关系维护。
2. 关键节点触达比无差别频繁触达更有价值。

## Part 2：未来业务蓝图

### Slide 15：Part 2 总览：公司 Agent + 个人 Agent + 价值交换

核心信息：

未来业务蓝图由三层组成：

1. 中心化公司 Agent。
2. 顾问个人 Agent。
3. 公司与个人之间的数据、服务和贡献价值交换。

讲述重点：

1. 公司 Agent 沉淀组织级资产。
2. 个人 Agent 放大顾问能力。
3. 两端通过代币、贡献积分、数据同步和权限隔离形成业务闭环。

### Slide 16：中心化公司 Agent：组织级能力中枢

核心信息：

公司 Agent 应包含：

1. 渠道寻访服务。
2. 线索池管理。
3. 公司人才库。
4. 公司知识库。
5. 飞书协作空间。
6. 外部动态获取。

讲述重点：

1. 公司 Agent 是组织级能力中枢，不只是工具集合。
2. 它统一承载服务、数据、知识和协作连接。
3. 它需要持续沉淀岗位画像模板、匹配评分模型、推荐策略资产和关联洞察资产。

### Slide 17：顾问个人 Agent：个人工作台与能力放大器

核心信息：

个人 Agent 应包含：

1. 轻量级个人人才库。
2. 个人定制化工具包。
3. 个人 know-how。
4. 推荐质量反馈。
5. 人选跟进工作台。
6. 数据与知识贡献。

讲述重点：

1. 个人 Agent 是顾问日常业务执行的工作台。
2. 个人 Agent 不应成为孤岛，而应把关键数据和知识沉淀到公司侧。
3. 个人 Agent 的价值是让顾问更快理解、更快执行、更持续地跟进。

### Slide 18：业务场景一：岗位咨询与寻访计划

核心信息：

流程：

```text
顾问提问 -> 公司 Agent -> 岗位画像 / 推荐人才画像 / 寻访策略
```

输出：

1. 岗位画像。
2. 推荐人才画像。
3. 人才寻访策略。
4. 决策辅助信息。

讲述重点：

1. 顾问不再从零开始理解 JD。
2. 公司 Agent 将过往知识、人才库和行业信息转化为可执行寻访计划。

### Slide 19：业务场景二：多渠道寻访与线索跟进

核心信息：

流程：

```text
公司寻访服务 -> 人才线索 -> 顾问跟进 -> 个人库 -> 公司库
```

同步内容：

1. 联系方式。
2. 跟进日志。
3. 备注。
4. 状态。

讲述重点：

1. 公司提供寻访能力，顾问负责高价值跟进。
2. 个人库是本地工作台，公司库是组织级资产。
3. 同步规则要统一，同时保持顾问间信息和数据隔离。

### Slide 20：业务场景三：推荐反馈与精准推荐优化

核心信息：

流程：

```text
推荐人选 -> 质量反馈 -> 评分模型 / 画像模板 -> 精准推荐提升
```

优化对象：

1. 人选匹配评分模型。
2. 岗位画像模板。
3. 推荐策略。

讲述重点：

1. 推荐不是一次性输出，而是一个可学习的反馈闭环。
2. 顾问反馈是模型优化和岗位画像优化的核心业务信号。

### Slide 21：业务场景四：联系人线索价值交换

核心信息：

流程：

```text
贡献联系方式 -> 贡献积分 -> 积分转代币 -> 购买联系方式
```

规则：

1. 联系方式交易不排他。
2. 一个顾问可以购买某个人选联系方式，其他顾问也可以购买。
3. 联系方式不是独占资产。
4. 第一个贡献有效联系方式者获得奖励。
5. 后续重复贡献不获得同等奖励。

讲述重点：

1. 个人贡献的数据通过积分变成可消费权益。
2. 公司人才库通过交易机制持续积累联系方式资产。

### Slide 22：业务场景五与六：洞察驱动深度跟进和长期关系维护

核心信息：

场景五：行业洞察与人选深度跟进。

```text
行业资讯 / 企业动态 / 领域进展 -> 关联人选信息 -> 推送顾问
```

场景六：深度跟进 SOP 与自动化触达。

```text
跟进 SOP / 自动化触达 -> 回访 / 情绪价值 / 长期关系维护
```

讲述重点：

1. 公司 Agent 通过外部动态和人选信息关联，生成可行动洞察。
2. 顾问基于洞察进行深度跟进。
3. 系统不只在有职位时联系人选，也在合适时机维护长期关系。

### Slide 23：价值交换与治理规则

核心信息：

代币：

1. 服务支付单位。
2. 用于消费公司 Agent 服务、数据、信息、联系方式、洞察和推荐能力。
3. 未来可演进为个人账户和支付体系。

贡献积分：

1. 数据价值度量单位。
2. 衡量联系方式、跟进数据、质量反馈、个人 know-how、知识总结和可复用经验。
3. 可对应积分或酬劳回报。

积分转代币：

1. 个人贡献获得积分。
2. 积分按规则转为代币。
3. 顾问再用代币消费公司服务和公司数据。

治理规则：

1. 顾问不能直接查看彼此完整工作过程。
2. 数据共享通过公司 Agent、人才库规则、积分和代币机制完成。
3. 成单归属和推荐机会归属先到先得。
4. 同一人选允许多人跟进。
5. 联系方式交易不排他。
6. 首个有效贡献者获得奖励。

讲述重点：

1. 没有价值交换机制，顾问不会持续贡献高质量数据。
2. 没有治理规则，顾问之间会产生归属冲突。
3. 代币和贡献积分是公司 Agent 体系商业化和内部激励的基础。

### Slide 24：未来业务闭环：从岗位理解到关系维护

核心信息：

五个闭环：

1. 岗位理解闭环：岗位咨询、岗位画像、寻访策略、推荐计划。
2. 寻访交付闭环：公司 Agent 寻访、顾问跟进、个人库记录、公司库同步。
3. 推荐优化闭环：推荐人选、顾问反馈、评分模型优化、画像模板优化、精准推荐提升。
4. 价值交换闭环：个人贡献、贡献积分、积分转代币、代币消费公司服务和数据。
5. 关系维护闭环：外部动态、关联洞察、深度跟进、SOP 触达、长期人选关系维护。

讲述重点：

1. 未来业务蓝图的本质是把个人能力变成公司级复利资产。
2. AI 的价值不是单点自动化，而是让业务闭环可持续学习和增长。

## Part 3：OPC 猎头公司设想

### Slide 25：Part 3 总览：从公司 Agent 到 OPC 一人猎头公司

核心信息：

OPC 一人猎头公司的设想，是用 AI 和公司级资产把一个顾问的经营能力放大。

讲述重点：

1. OPC 不是一个人做所有事。
2. OPC 是把低价值事务 AI 化，把人的时间集中在高信任、高判断、高成交环节。
3. 公司 Agent 和个人 Agent 的体系，是 OPC 模式成立的能力基础。

### Slide 26：OPC 收益目标：3125W 人选薪资对应 500W 年收益

核心公式：

```text
3125W 人选薪资 × 20% 服务费率 × 80% 个人留存 = 500W 年收益目标
```

补充说明：

1. 这里是 OPC 成型后的收益公式。
2. `3125W` 是人选薪资现金口径。
3. `20%` 是服务费率。
4. `80%` 是个人留存或提点口径。
5. 目标是实现 `500W` 年收益。

讲述重点：

1. OPC 模式的核心是高价值成交能力与高留存结构。
2. 这个公式用于定义未来目标状态，而不是当前公司提点规则。

### Slide 27：当前公司口径下的难度参照：24 个年包 300W+ 人选

核心信息：

当前公司口径下，约等于完成 `24 个 / 年` 的年包 `300W+` 高端人选成交。

说明：

1. 这是难度参照。
2. 不套用 OPC 公式中的 `20%` 服务费率和 `80%` 留存。
3. 目的是把未来 `500W` 年收益目标，转换成今天可以感知和管理的成交任务量。

讲述重点：

1. 当前口径用于评估可行性和难度。
2. OPC 目标不是抽象愿景，而可以被拆解成高端人选成交任务。

### Slide 28：OPC 交付漏斗：从 AI 寻访到成单交付

核心信息：

年度目标对应的交付漏斗：

1. 寻访 AI：约 `150/天`，时间投入接近 `0%`。
2. 外联打通：约 `10/天`，时间投入约 `55%-60%`。
3. 推荐高质量：约 `6/周`，人工投入 `<5%`。
4. 面试推进：约 `2/周`，时间投入约 `5%`。
5. 成单交付：约 `2/月`，时间投入约 `10%-15%`。

讲述重点：

1. 人的主要时间应放在外联打通、关系经营、关键判断和成交闭环。
2. AI 承担寻访、材料、录入等事务性工作。
3. 高质量推荐和成单推进是漏斗质量的关键。

### Slide 29：OPC 长期资产：可复用经营资产持续增长

核心信息：

OPC 需要持续沉淀四类资产：

1. 深度人脉：高信任关系与长期跟进。
2. 人选库：可复用、可激活的人才资产。
3. JD 池：持续积累机会供给，约 `10%` 时间投入。
4. 客户池：降低未来获客成本，约 `5%-10%` 时间投入。

讲述重点：

1. OPC 不是只靠当下机会成交，而是靠资产复利。
2. 深度人脉、人选库、JD 池、客户池共同决定未来获客和交付成本。

### Slide 30：OPC 时间分配：把人工时间集中在高杠杆动作

核心信息：

建议时间分配：

1. 外联打通：`55%`。
2. 成单：`12%`。
3. JD 池：`10%`。
4. 客户池：`8%`。
5. 面试：`5%`。
6. 材料系统：`5%`。
7. 缓冲：`5%`。

讲述重点：

1. OPC 的关键不是把所有环节都手工做完。
2. 应把人工时间集中在外联、关系经营、关键判断和成交闭环。
3. 低价值事务应由 AI 和系统承担。

### Slide 31：结尾：从 AI 猎头公司到高杠杆个人经营

核心信息：

这套体系最终连接三件事：

1. 公司级 AI Agent：沉淀组织资产、服务、知识和数据。
2. 顾问个人 Agent：放大个人执行、判断和关系维护能力。
3. OPC 经营模型：把顾问个人能力转化为高杠杆的一人猎头公司。

讲述重点：

1. AI 猎头公司的价值不是短期自动化，而是构建长期可复利的业务系统。
2. 当公司 Agent 和个人 Agent 形成闭环，顾问可以获得接近 OPC 的经营杠杆。
3. 未来的竞争优势来自数据资产、反馈闭环、深度人脉和高质量成交能力。
